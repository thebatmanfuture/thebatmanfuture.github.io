    查杀思路

    1）查看所有内建模块

    warnings.catch_warnings() 和 site.Quitter() 都是 Python 内建模块中的函数，warnings.catch_warnings() 函数属于 warnings 模块，提供了一种方便的方法来忽略特定类型的警告，并在执行结束后恢复警告设置。

    一般情况下，程序为了安全，不会在内建模块中允许eval，exec这张代码执行的函数存在，而如果通过模板注入实现内存马注入之后，调用了os模块内的eval，可以作为查杀参考

    2）检测self.add_url_rule()中特殊名字的路由如shell等

    这个关系不大

    3）检测文件的请求头和日志

    如果攻击者访问，并且频繁访问/shell路由，并且你的程序中并没有这个方法，那大概率就是内存马

    JAVA内存马

    如图，java内存马大体分为三类：

    servlet-api类

    filter型

    通过反射注入了一个名为 n1ntyfilter 的 Filter,当用户发送带有 cmd 参数的 HTTP 请求时，该 Filter 会执行 /bin/bash -c 命令，并将结果返回给客户端。

    servlet型

    创建了一个 Servlet 对象，并尝试将其封装成一个 Wrapper 对象，以便能够被 Tomcat 容器识别和调用。然后，该页面将封装后的 Wrapper 添加到当前 StandardContext 的 children 中，并通过 addServletMapping() 方法将 URL /shell 与该 Servlet 进行绑定，在注入后，如果用户访问 /shell URL 并传递合法的 cmd 参数，则该恶意 Servlet 将执行相应的操作，并将结果返回给客户端。具体而言，它通过调用 Runtime.getRuntime().exec() 方法来执行指定的系统命令，并将命令输出发送给客户端。

    listener型

    使用 JSP 页面实现了一个 ServletRequestListener 注入的过程。具体而言，它通过反射机制获取当前 ServletContext 对象，并进一步获取到当前 Web 应用程序的 ApplicationContext 和 StandardContext 对象。然后，该页面创建了一个名为 listenerdemo 的 ListenerDemo 对象，并将其添加到 StandardContext 中，从而实现了 Listener 注入。

    在注入后，如果用户发送带有 cmd 参数的 HTTP 请求，则该 ListenerDemo 对象会从 ServletRequestEvent 中获取到请求参数，并直接调用 Runtime.getRuntime().exec() 方法来执行相应的系统命令。由于该 Listener 是全局性的，并且能够监听并处理所有的 HTTP 请求事件，因此攻击者可以通过构造特定的请求，轻易地将任意的系统命令发送给服务器。
